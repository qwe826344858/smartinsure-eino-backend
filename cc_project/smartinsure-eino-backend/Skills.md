# 多Agent开发任务执行器

## 使用场景
当需要执行复杂开发任务时使用，例如：

- 对大型功能模块进行拆分开发
- 需要多模块并行开发的场景
- 需要独立验证的开发任务
- 开发过程中可能出现异常需要确认的情况

## 任务拆分
- Team Leader 接收任务后自动拆分为独立模块
- 每个模块分配给独立的 Developer Agent
- 模块间依赖关系自动识别和管理

## 并行开发
- 多个 Developer Agent 同时执行不同模块
- Team Leader 实时监控各 Agent 进度
- 任务执行到关键阶段时反馈任务进展

## 独立验证
- 所有模块开发完成后触发 Verifier Agent
- Verifier Agent 完全独立，不参与开发过程
- 执行代码审查、功能测试、集成验证

## 异常确认
- Developer Agent 遇到无法解决的异常时主动上报
- Exception Handler 向用户展示问题详情和建议选项
- 用户确认后 Agent 继续执行或调整方案

## 任务开始前
- 当Team Leader 创建任务时，最开始先询问是否需要读取上次未完成的文档，如果需要则通过输入的文档名称进行加载，如果不需要则自动生成新的任务文档
- 任务执行过程中，每个模块执行进度自动更新文档
- 任务完成后，自动生成任务报告；任务遇到异常问题时，自动生成异常报告要声明对应Agent执行的模块名称和异常信息
- 任务报告包含任务执行结果、异常情况、建议及解决方法

## 文档相关
- 文档内容包含：Agent列表、任务名称、任务描述、模块列表、模块依赖关系、任务执行流程、任务执行要求、任务执行步骤、任务执行结果、异常情况、建议及解决方法
- 文档生成时，自动生成时间戳
- 文档生成时，自动生成任务编号
- 文档生成时，自动生成任务状态
- 文档生成时，自动生成任务进度
